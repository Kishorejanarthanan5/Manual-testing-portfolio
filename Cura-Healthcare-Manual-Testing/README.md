# CURA Healthcare – Manual Testing Project

This repository contains a complete manual testing portfolio for the **CURA Healthcare Service** application.  
The project is designed to demonstrate structured manual testing practices followed in enterprise QA environments.

---

## Application Under Test

**CURA Healthcare Service**  
A healthcare appointment booking web application that allows users to log in, book appointments, view appointment history and manage sessions.

---

## Tools Used

- **Jira** – Defect tracking, issue management, and test management
- **Zephyr** – Test case management, test cycle creation, and execution tracking
- **GitHub** – Version control and repository for test artifacts and documentation

---

## Test Artifacts Included

- Test Plan defining scope, test strategy, and entry/exit criteria
- Test Scenarios and detailed Test Cases
- Test Execution results using Zephyr
- Defect Reports logged and tracked in Jira
- Requirement Traceability Matrix (RTM) for test coverage analysis
- Jira and Zephyr execution evidence (screenshots)

---

## Test Scope

- Login functionality
- Appointment booking workflow
- Appointment confirmation
- Logout and session handling

---

## QA Activities Performed

- Requirement analysis based on application functionality
- Preparation of Test Plan outlining scope, approach, and test types
- Test scenario creation covering positive, negative, and edge cases
- Detailed test case design using structured templates
- Manual test execution using Zephyr (integrated with Jira)
- Defect logging, tracking, and status updates in Jira
- Requirement Traceability Matrix (RTM) preparation to ensure test coverage

---

## Evidence

Screenshots from Jira and Zephyr are included to demonstrate test case execution, test cycle progress, and defect management activities.

This project reflects hands-on experience working with structured QA processes and industry-standard testing tools.
